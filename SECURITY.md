# Security Policy

## Supported Versions

The following versions of CertGuard receive security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

Only the latest minor release within a supported major version receives patches.
Once a new minor version is released, the previous minor version is no longer supported.

## Reporting a Vulnerability

If you discover a security vulnerability in CertGuard, please report it responsibly.
**Do not open a public GitHub issue for security vulnerabilities.**

To report a vulnerability:

1. Email the maintainers with a description of the issue, steps to reproduce, and any
   relevant logs or proof-of-concept code.
2. You will receive an acknowledgement within 48 hours.
3. The team will investigate and provide an estimated timeline for a fix.
4. Once a fix is available, a security advisory will be published and a patched release issued.

Please include the following in your report:

- Description of the vulnerability
- Affected version(s)
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We appreciate responsible disclosure and will credit reporters in the release notes
(unless you prefer to remain anonymous).
